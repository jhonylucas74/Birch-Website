Mas o que é isso?

Birch é um gerador de classes java, mais especificamente um gerador de Data Access Object, conhecido popularmente como DAO. O objetivo principal é a redução do tempo de desenvolvimento de um DAO. Tornando simples o processo de criação, fazendo algo que demoraria alguns minutos para alguém experiente, em apenas alguns segundos. Sua aplicação é tanto profissional como didática. Auxiliando o seu utilizador ao acelerar a concepção de um DAO tradicional, até como exemplo didático para aqueles que tem dificuldades ou está iniciando seus estudos sobre banco de dados e java.

Inicialmente birch foi projetado para ser utilizado no ubuntu, no entanto é possível utilizá-lo em outros S.O’s e distribuições do linux. Abaixo segue dois links que descrevem como deve ser realizado o processo de “instalação” para o linux e windows.

Comandos no birch

Birch foi feito para ser um programa executado por linhas de comando, algumas pessoas não estão muito acostumados com isso. Sem interface gráfica a única coisa que precisa fazer é decorar alguns comandos, ou favoritar a página e quando tiver dúvida dar um pulinho aqui. Logo após a configuração adequada do birch no seu S.O é esperado que ele esteja acessível pelo seu terminal de comando. Vamos levar em consideração que já temos um projeto no eclipse e queira criar um DAO para uma classe chamada Aluno, sendo que essa classe ainda não existe. Abra o seu terminal e entre na pasta do seu projeto, comigo seria assim.

$ cd workspace/
workspace$ cd Escola
workspace/Escola$ cd src/
workspace/Escola/src$

Agora Estou dentro da pasta Escola que foi o nome que dei para o meu projeto. Para criar meu DAO escrevo o seguinte comando.

workspace/Escola/src$ birch Aluno nome:varchar\(50\) media:real idade:int

Aperto enter e a saída obtida deve ser algo parecido com isso:

=====================
Analisando parâmetros e criando as classes java:

ConnectionFactory.java
AlunoDAO.java
Aluno.java
Tables.java

Todos os arquivos foram criados com sucesso.

Agora eu já tenho o meu DAO pronto, foram criados quatro arquivos: AlunoDAO.java que é o DAO em se, sem mistério nenhum. Aluno.java que é minha classe Aluno que possui os atributos convertidos, e já vem pronta para ser integrada junto com o meu programa. A ConnectionFactory que é uma classe padrão utilizada pelos DAO’s. Tables.java é uma classe que pode ser utilizada para a criação das tabelas do BD.

Pelo que podemos ver, criar um DAO é algo fácil. Cuidado, ele sempre criará esses quatro arquivos. Então caso você já tivesse uma classe chamada Aluno.java ela seria substituída. Agora vamos entender as partes do comando executado. Você pode passar quantos atributos você quiser para a criação do seu DAO. No comando a primeira coisa logo após o nome birch é o nome do modelo que queremos criar, que no caso seria Aluno. Esse será o nome da minha classe java, e o nome da minha tabela(entidade). Caso queira que sua entidade tenha um nome diferente, basta você escrever o comando dessa maneira:

workspace/Escola/src$ birch Aluno -d Alunos nome:varchar\(50\) media:real idade:int

Utilizando -d podemos dar um nome diferente para a entidade relacional, bastando somente colocar o nome da entidade logo após o -d. Em seguida percebe-se que ao adicionar os atributos do modelo, definimos o tipo igualmente como definiríamos utilizando sql. Detalhe, ao usar parênteses é necessário utilizar uma contra barra. Os tipos de atributos aceitos são: char, varchar, int, smallint, float, real.

Também é possível utilizar a opção -id. Ao usá-lo adicionamos um atributo chamado id no nosso schema, ele é definido como chave primária. Toda vez que o DAO salvar uma instância do objeto, seu id será gerado automaticamente.

workspace/Escola/src$ birch Aluno -id nome:varchar\(50\) media:real idade:int
